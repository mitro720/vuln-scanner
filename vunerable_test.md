Of course! Since this is for your project presentation, you'll want to scan intentionally vulnerable sites that are legally safe and built for students/security researchers.

Here is a list of 10+ excellent test-bed sites that are perfect for your presentation because they generate high-severity findings (SQLi, XSS, etc.) that will make your dashboard and risk scores look very impressive:

1. The "VulnWeb" Labs (By Acunetix)
These are the gold standard for testing web scanners.

PHP: http://testphp.vulnweb.com
ASP.NET: http://testaspnet.vulnweb.com
ASP.old: http://testasp.vulnweb.com
REST API: http://rest.vulnweb.com (Great for your API discovery modules)
2. Financial Services Simulations
These simulate banking apps and have great "corporate" vulnerability UI potential.

Altoro Mutual: http://demo.testfire.net (Very classic, highly vulnerable)
Zero Bank: http://zero.webappsecurity.com
3. Educational Vulnerable Apps
OWASP Juice Shop: http://juice-shop.herokuapp.com (Modern Javascript shop, shows off SSTI and NoSQL injection)
HackyYourselfFirst: http://hackyourselffirst.troyhunt.com (By Troy Hunt, covers many modern security headers)
WebScanTest: http://webscantest.com (Designed specifically for automated tools)
CrackMe: http://crackme.cpro-it.com (General vulnerability set)
4. Modern Injection Targets
VulnNodeApp: http://vuln-node-app.herokuapp.com (If you want to test Node.js specific attacks)
